Technical Events

Click the event name for Rules
  • PAPER PRESENTAION

    Candidate has to submit the abstract by 25th of july
    There can be two or three members per team
    The presentation should have 5 to 7 minutes of duration
    The winners wil be choosed by the judge and the decision will be final.
    The topics should be related to new technology in computer science field.

    Topics:
    Big data
    Data mining
    Internet of things
    Grid computing
    Network security

  • DEBUGGING

    The event contains 2 rounds in total.
    First round contains preliminary round.
    A set of questions related to C programming will be given and candidates will have to solve them in the given time.
    The second round is the final one, where the candidate needs to debug the given code, compile and run the program.
    The candidate who completes the debugging at the shortest span of time will be the winner.
    The winner and runner will be chosen strictly on the basis of the given set of rules.

  • WEB DESIGNING

    This event contains 2 rounds in total.
    First round contains prelimanary round. In this round you need to answer 60 questions in 30 minutes.
    From the first round top 10 scorers will be selected.
    The second round is the final round.
    The candidates should create a webpage using the provided source and should complete the given task in one hour.
    The theme for creating the webpage will be provided on the spot.
    The winner and the runner will be selected according to the theme, efficient use of source and runtime of the website.

  • CODE PUZZLE

    Round 1:
    Two memebers in a team.
    Three questions for a team.
    Questions are based on C , java and html.
    Each question carries 10 points and if a question is not answered then it will become an open question.
    Open question carries five points but if the answer is wrong then five points will be reduced.
    Top seven teams will move to the final round.

    Round 2:
    This round will have seven teams.
    Each team will be given an envelop, which contains pieces of a program in a shuffled manner.
    The main objective of this round is to arrange them in a proper order.
    The first two teams to finish arranging the program will be declared the winners and runners respectively.

  • WORD HUNT

    Three members per team.
    Each team should hunt for three technical words.
    One member can pick up alot and the other member should find the word by using the clues.
    The clues should be from hardware or software.
    The lot members should answers only “yes” , “no” or “maybe”.
    Using other words or signs will disqualify the team.
    60 seconds will be given for each hunt.
    Once the word is found, the remianing seconds will be taken as the score for the hunt.
    Decision of the juries will be finalized.
